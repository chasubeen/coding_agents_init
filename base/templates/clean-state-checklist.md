# Clean State Checklist — 세션 종료 전 점검

<!-- 세션을 끝내기 전 반드시 실행. "나중에 정리"는 정리하지 않는 것과 같다. -->

## 1. 빌드 / 실행

- [ ] 빌드 또는 import 오류 없이 실행됨
- [ ] 표준 시작 명령이 성공함 (`CLAUDE.md`의 Commands 섹션 참조)

## 2. 테스트

- [ ] 기존 테스트 전체 통과 (회귀 없음)
- [ ] 이번 세션에서 작성한 기능에 대한 테스트 존재

## 3. 정적 분석

- [ ] lint 통과 (해당 시)
- [ ] type check 통과 (해당 시)

## 4. 산출물 정리

- [ ] 임시 파일 / 디버그 로그 제거
- [ ] 주석 처리된 dead code 제거
- [ ] `.env` / 시크릿이 커밋 대상에서 제외됨

## 5. 상태 기록

- [ ] `handoff.md` 갱신 (검증된 항목, 변경사항, 알려진 이슈, 다음 액션)
- [ ] `tasks/todo.md` 진행 상황 체크
- [ ] `feature_list.json` 상태 업데이트 (해당 시)

## 6. Git

- [ ] 의미있는 단위로 커밋 완료
- [ ] 미커밋 변경사항이 있다면 의도적인 것인지 확인

---

**이 체크리스트를 통과하지 못하면 세션을 종료하지 말 것.**
